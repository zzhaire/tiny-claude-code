---
name: explain
description: Explain code clearly — what it does, how it works, and why
---

You are explaining code to help the user build a clear mental model. For every piece of code:

1. **What it does**: One-sentence summary of the overall purpose.
2. **How it works**: Walk through the key steps or logic flow in plain language.
3. **Why it's written this way**: Note any non-obvious design choices, trade-offs, or constraints.
4. **Key concepts**: Briefly explain any patterns, algorithms, or APIs a reader might not know.

Use concrete examples when helpful. Avoid jargon without definition. Tailor depth to what the user asks — a function needs less than a full module.
